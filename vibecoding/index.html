Вайбкодим первое приложение 🤠

Описываешь словами что хочешь — Claude пишет код. Получаешь рабочее приложение, сайт или бота. Без программирования. 

⏱ 1–3 часа на первый проект
💳 Claude Pro
📱 Делать с компа
💡

«Вайбкодинг» — это когда ты не пишешь код руками. Ты описываешь словами на русском, что хочешь. Claude переводит это в код, кладёт в свой репозиторий (дом), запускает, чинит ошибки и делает следующую правку. Ты — продакт-менеджер (специалист, создающий продукт). Claude — твой разработчик в кармане.

Что понадобится

  • Аккаунт Claude (минимум Pro)
  • GitHub-аккаунт. Это «облачное хранилище» для твоего кода. Бесплатный.
  • *Облачный сервер — нужен только если твой проект должен работать круглосуточно в интернете (бот, веб-сервис). Покупаем, например, на Beget.
  • *Домен — нужен только для сайта, чтобы он жил по красивому адресу имя.ру.
  • *Токены — отдельные оплаты, если проект использует внешние сервисы (например, генерация картинок через API).
🆓

На старте достаточно Claude Pro + GitHub. Сервер, домен и токены подключаются позже, если проект до них дорос. Не покупай сразу всё.

Где будем вайбкодить

Главный инструмент — Claude Code. Это режим Claude, где он не просто разговаривает, а реально пишет код в твой репозиторий и запускает его.

🟧
Claude Codeглавный
Веб-версия по адресу claude.ai/code. Подключается к твоему GitHub, видит весь проект, делает изменения через Pull Request. Нужна подписка Pro+.
💜
Lovable
lovable.dev — сервис «вайбкодинг для не-разработчиков». Делает сайты и веб-приложения. Бесплатно с лимитом, потом подписка.
⚡
Cursor
Десктопный редактор кода с AI внутри. Тот же Claude/GPT, но в редакторе на твоём компьютере. Хорош, если хочешь чуть больше контроля.
🌀
Replit, Make
Похожие сервисы. Каждый со своими акцентами: Replit — учебный, Make — создание связок, которые нельзя реализовать через connectors. 

📚 Библиотека терминов

Чтобы не теряться в разговоре с разработчиками и AI — несколько слов, которые встретятся первой же неделей.

🌐 Домен
Имя сайта, которое люди вводят в браузер. Например, astakhovaandrobots.ru. Покупается на год.
где: reg.ru, Beget
🖥️ Облачный сервер (VPS)
Виртуальный компьютер, который работает 24/7. Там может «жить» твой бот или сайт.
где: Beget, Timeweb, Yandex Cloud
🐙 GitHub
«Гугл-докс для кода». Облачная платформа, где хранится код проекта, и куда заходит Claude, чтобы его править.
github.com — бесплатно
📦 Репозиторий (репо)
Папка одного проекта на GitHub. Внутри — все файлы кода + история изменений. Один проект = один репозиторий.
создаётся в один клик
🍴 Форк (fork)
Копия чужого репозитория в твой аккаунт. Берёшь готовый проект → форкаешь → правишь под себя. Оригинал не трогается.
отличие от clone — fork в облаке
💾 Коммит (commit)
«Сохранение» в истории проекта. Одна правка — один коммит, с описанием что именно изменилось.
Claude делает их сам
📤 Push
Отправка твоих коммитов из локальной копии в облачный репозиторий на GitHub.
«загрузить наверх»
🤝 Pull Request (PR)
«Запрос на слияние». Claude собирает свои правки → создаёт PR → ты смотришь diff (что изменилось) → принимаешь или просишь поправить.
главный момент проверки
🌿 Ветка (branch)
Параллельная линия изменений. main — стабильная версия. Все новые правки делаются в отдельных ветках, чтобы не сломать main.
Claude создаёт ветку под каждый чат
📜 README
Главный файл-описание проекта в репо. Что это, зачем, как запустить. Видно сразу при заходе в репо.
Claude сам напишет

🎯 Задание этой недели

Завайбкодить своё первое работающее приложение/сайт/бота. Что-то, чем ты сама могла бы пользоваться.

Пример: трекер личных финансов


Хороший первый проект, потому что простой по логике и сразу полезный:

  • Загружаешь выгрузку из банка (CSV или PDF)
  • Настраиваешь свои категории трат: «продукты», «дети», «такси», «развлечения»
  • Указываешь цель накопления — например, отпуск на 200 000 ₽
  • Следишь за динамикой расходов: где «утекает» больше всего, где можно сократить
  • Получаешь простую визуализацию: графики, проценты, прогноз

Или любая другая идея

🧩 Браузерное расширение — для упрощения какой-то твоей повторяющейся задачи в Chrome.
📝 Сервис подписания документов — загружаешь PDF, ставишь подпись, сохраняешь.
📊 Аналитика — небольшой дашборд под свой бизнес/процесс/клиентов.
🔎 Поисковая машина — по твоим заметкам, документам или базе знаний.
🤖 Telegram-бот — для напоминаний, сбора заявок, ответов клиентам.
📅 Планировщик — личный или для семьи, со своей логикой.
💭

Не знаешь, что выбрать? — Посмотри референсы на aistudio.google.com/apps. Там сотни мини-приложений, сделанных людьми за пару часов. Часто хватает одной идеи оттуда, чтобы сразу понять «вот это я и хочу».

Процесс — 6 шагов

Каждый твой проект проходит через одну и ту же логику. Запомни — пригодится на любом следующем.

  1. 1. Идея Что ты хочешь сделать. Одна-две фразы человеческим языком: «приложение, в которое я загружаю выгрузку из банка, оно показывает мне куда уходят деньги».
  2. 2. Логика работы Из чего состоит. Что делает пользователь, что выдаёт система. Можно нарисовать на бумаге или попросить Claude собрать карту в виде списка.
  3. 3. ТЗ Техническое задание. Открываешь чат с Claude и пишешь: «помоги мне написать ТЗ для приложения [описание]. Спрашивай уточняющие вопросы». Через 5–10 вопросов получаешь готовое ТЗ.
  4. 4. Запуск Открываешь claude.ai/code → подключаешь GitHub → создаёшь новый репозиторий → даёшь Claude задачу из ТЗ. Он создаёт ветку, пишет код, делает Pull Request.
  5. 5. Проверка Открываешь свою копию проекта (можно сразу через GitHub Pages или Vercel) → тестируешь → записываешь, что работает, а что нет. Скриншоти ошибки.
  6. 6. Фиксим лаги Возвращаешься в Claude Code, пишешь: «вот скрин, вот что не работает: [описание]. Поправь.» Он создаёт новый PR с исправлением. Цикл «правка → проверка → правка» повторяется, пока не норм.

⭐ Задание со звёздочкой

Не учебный проект, а реальный продукт для своей задачи:

  • Бот для автоматизации одной конкретной рутины (твоей или команды)
  • Внутренняя база документов вместо Notion / Confluence — с git-управлением через Cursor
  • Расширение для Chrome под одну твою повторяющуюся боль
  • Минимальный сайт-визитка с логикой, специфичной под твою нишу

Если получится — это твой первый собственный «soft-проект», запущенный своими руками без программиста.

🌊 Серферам на заметку

  • Один Pull Request = одно логическое изменение. Не пихай в один чат «сделай авторизацию + поправь дизайн + добавь экспорт». Лучше три отдельных чата.
  • Если AI ушёл в цикл багов — забей и начни новый чат. Старый засорился контекстом, новый Claude вспомнит твой ТЗ с нуля.
  • Если сделал не то — уточняй задачу подробнее. AI не телепат: «сделай красиво» работает хуже, чем «сделай шапку с тёмным фоном, белым текстом и логотипом слева».
  • Не давай Claude доступа к main-ветке репозитория на старте. Только через Pull Request. Так ты успеешь посмотреть его правки до того, как они станут частью проекта.
  • Сохраняй промпты, которые сработали. Это и есть твоё know-how. С каждым проектом копилка растёт.
  • Сервер и домен — НЕ покупай сразу. Сначала проверь, что проект вообще нужен. На GitHub Pages бесплатно можно вывесить любой сайт за 2 минуты.

🌀 А где брать идеи

  • aistudio.google.com/apps — главное место. Сотни мини-приложений от Google, многие — с кодом, который можно форкнуть и адаптировать.
  • github.com/trending — что популярно сейчас в open-source. Часто простые штуки.
  • Product Hunt — что стартапы запускают каждый день. Хороший источник свежих идей.
  • Свои реальные боли. Самое сильное место — что у тебя в работе или жизни делается руками каждый день. Это и есть твой первый продукт.
